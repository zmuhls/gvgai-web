# Inference Arcade UX Wireframe

## Review

The web app already has a strong core flow: choose a GVGAI game, give the model a strategy, then watch the move trace. The earlier interface treated that flow like a dark control panel, while the project concept works better as a readable arcade table where play and critique sit together.

The revised UX keeps the three-step path and shifts emphasis toward critical reading. Each screen asks the player to compare a stated reason with an observed move, which fits the project better than a pure benchmark dashboard.

## Screen Structure

1. Header
   - Brand: Inference Arcade
   - Trace readout: game state, model reason, move choice
   - Navigation: Arcade and Prompts

2. Choose a Game
   - Featured cabinets first
   - Full catalog behind the browse toggle
   - Thin card grid over the page grid
   - Matching links to Cloze Reader and Jeopardy LM as other inference arcade rooms

3. Set a Strategy Question
   - Selected cabinet callout
   - Four strategy prompts
   - Editable strategy text
   - Model and level controls

4. Run View
   - Game canvas as the main play surface
   - Small stat cluster inside the canvas edge
   - Move Trace panel beside the playfield
   - Run summary below the playfield

5. Prompt Workbench
   - Sidebar for games and templates
   - Main editor for system, game, and level context
   - Prompt preview remains in the same editing area

## Visual Direction

The theme uses a light gunmetal field, thin grid lines, dark ink text, teal action states, brass emphasis, and sparse glitch motion. The page should feel like a playable research cabinet: restrained enough for reading, and still close to an arcade surface.
