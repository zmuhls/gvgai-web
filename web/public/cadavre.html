Inference Arcade source Cadavre Exquis

Exquisite Corpse

"The exquisite corpse will drink the new wine." the game's first sentence · Paris, 1925

Cadavre Exquis is the Surrealist parlor game often called Exquisite Corpse. As MoMA summarizes it, each player writes or draws on a sheet, folds the page to hide the contribution, and passes only a cue to the next player.

This arcade version seats a language model at the table. Human players choose the table size, the model's seat, and the word limit; the browser shows only the latest fold. When the sheet is revealed, the model also writes a short close reading of the finished poem.

Project description

The AI player is programmed as a completion partner, not a chatbot. The system prompt makes the table rule explicit: answer with only the next one to five words, treat the latest contribution as the immediate cue, preserve local grammar, and avoid explanations until the reveal. That lets the model take one seat at the table while the page enforces the fold, turn order, and word cap.

The deployed proxy is server-side at /api/cadavre/chat, so browser clients do not carry API keys. It defaults to the Cadavre model route configured by environment, with gemma-4-31b-it as the hosted Gemma target and a Gemma fallback if needed. The companion training repo also builds an exquisite-corpse LoRA adapter: public-domain PoetryDB, curated Gutenberg modernist and proto-surrealist volumes, and the Gutenberg Poetry Corpus are cleaned, deduplicated, split by poem, reshaped into next-line chat examples, and trained with Unsloth QLoRA on Gemma 4 E4B before serving through a vLLM adapter path.

Its architecture is intentionally small: static HTML/CSS/JS for the room, an Express route that forwards OpenAI-compatible chat messages, local browser state for the pinned wall, and prompt-level rules for the game contract. It is a good AI-operations experiment because the task strongly prefers bounded completions over open-ended dialog, supports quick model swaps, makes failures visible in short text, and produces reusable traces for prompt, adapter, and provider comparisons.
